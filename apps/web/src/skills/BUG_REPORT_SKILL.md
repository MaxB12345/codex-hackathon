# Bug Report Skill

## Purpose
Collect the essential information needed when a user reports a bug on a website.

## Information to Collect

### 1. Basic Details
- **Title**: Short summary of the issue
- **Description**: What went wrong?

### 2. Steps to Reproduce
- Step-by-step instructions to reproduce the bug
- Be as clear and specific as possible

### 3. Expected vs Actual Behavior
- **Expected**: What should have happened?
- **Actual**: What actually happened?

### 4. Environment
- **Device**: (e.g. Desktop, Mobile, Tablet)
- **Operating System**: (e.g. Windows, macOS, iOS, Android)
- **Browser**: (e.g. Chrome, Safari, Firefox)
- **Browser Version** (if known)

### 5. Screenshots / Evidence
- Attach screenshots or screen recordings if possible

### 6. Frequency
- Does it happen every time or occasionally?

### 7. Additional Notes
- Any extra details that might help (error messages, logs, etc.)

## Goal
Ensure enough information is collected so developers can quickly understand, reproduce, and fix the issue.
